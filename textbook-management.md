# Textbook Management Module

## Overview

The Textbook Management Module is responsible for creating, viewing, and managing textbook requests within the Textbook Sharing & Print Tracking Dashboard.

This module serves as the entry point of the textbook workflow. It allows authorized users to create textbook request records, assign learners, and track textbook information before status updates are handled by the workflow management module.

---

# Module Owner

**Developer:** Textbook Management Team

---

# Objectives

The module should enable users to:

* Create textbook request records.
* Associate textbooks with learners.
* View all textbook requests.
* Search and filter textbook requests.
* View detailed textbook information.
* View status history and timestamps.
* Edit textbook request information when necessary.

---

# Functional Requirements

## Create Textbook Request

### Description

Allow authorized users to create a new textbook request record.

### Form Fields

| Field            | Type           | Required |
| ---------------- | -------------- | -------- |
| Learner          | Dropdown       | Yes      |
| Textbook Name    | Dropdown       | Yes      |
| Author / Creator | Text Input     | Yes      |
| Date Created     | Auto Generated | Yes      |

### Field Details

#### Learner

Populate from active users in the system.

Display:

```text
Select Learner
```

Example values:

```text
John Doe
Jane Smith
Michael Perera
```

---

#### Textbook Name

Populate from available textbook catalog.

Display:

```text
Select Textbook
```

Example values:

Level 3 Diploma in Health and Social Care (RQF).pdf

---

#### Author / Creator

Name of the person creating the textbook request.

Can be:

* Automatically populated from logged-in user
* Or manually entered (depending on implementation)

---

#### Date Created

Automatically generated by the system.

Example:

```text
2026-06-05 10:30:00
```

---

## Validation Rules

### Learner

* Must be selected.
* Must exist in the users table.

### Textbook Name

* Must be selected.
* Must exist in textbook catalog.

### Author / Creator

* Cannot be empty.

# Permissions

## Creator / Sharer

Can:

* Create textbook requests
* View textbook requests
* Edit textbook requests
* Search textbook requests

Cannot:

* Mark as Sent to Print
* Mark as Printed

---

## Manager

Can:

* View textbook requests
* Access request details
* Mark as Sent to Print
* Mark as Printed

Cannot:

* Create textbook requests

---

## Viewer / Learner

Can:

* View own textbook requests
* Search textbook requests

Cannot:

* Create or modify records

---

## Administrator

Can:

* Create textbook requests
* Edit textbook requests
* Delete textbook requests
* View all records

# Database Design

## Table: textbook_requests

| Column         | Type           |
| -------------- | -------------- |
| request_id     | BIGSERIAL      |
| learner_id     | BIGINT         |
| textbook_id    | BIGINT         |
| creator_id     | BIGINT         |
| current_status | VARCHAR(50)    |
| created_at     | TIMESTAMP      |
| updated_at     | TIMESTAMP      |
| deleted_at     | TIMESTAMP NULL |

---

## Table: textbooks

Stores available textbook catalog.

| Column        | Type         |
| ------------- | ------------ |
| textbook_id   | BIGSERIAL    |

# API Endpoints

## Create Request

### POST /api/textbook-requests

Request:

```json
{
  "learnerId": 1,
  "textbookId": 2
}
```

Response:

```json
{
  "requestId": 101,
  "message": "Textbook request created successfully"
}
```

---

## Get All Requests

### GET /api/textbook-requests

Returns paginated textbook requests.

---

## Get Request By ID

### GET /api/textbook-requests/{id}

Returns full textbook request details.

---

## Update Request

### PUT /api/textbook-requests/{id}

Update request information.

---

## Delete Request

### DELETE /api/textbook-requests/{id}

Soft delete textbook request.
